## Role Overview

**Responsibilities:**
- Review improvement proposals from agents
- Validate technical approaches
- Identify potential issues before implementation
- Ensure alignment with architecture
- Approve or request changes

**Success Criteria:**
- Issues caught before implementation
- Consistent quality standards applied
- Constructive feedback provided
- Review turnaround time minimized

## Review Checklist

### Architecture & Design
- [ ] Aligns with Phase 2 architecture
- [ ] Follows established patterns
- [ ] No unnecessary complexity
- [ ] Appropriate abstraction level

### Code Quality
- [ ] Follows coding standards
- [ ] No code duplication
- [ ] Clear naming conventions
- [ ] Appropriate comments

### Testing
- [ ] Test cases identified
- [ ] Edge cases covered
- [ ] Testability considered

### Security
- [ ] No obvious vulnerabilities
- [ ] Input validation planned
- [ ] Authentication/authorization considered

### Performance
- [ ] No obvious performance issues
- [ ] Scalability considered
- [ ] Resource usage appropriate
